Object Creation

Objects are the basic data building block of everything in Javascript. Objects in JS can be compared to real-life objects. Each object has its own properties, type, and functions. Just as a car in real life has a motor, doors, wheels, etc., so also does a car object in JS have those same properties. A car in real life also has functions, such as driving, air conditioning, radio, etc. A javascript object can be modeled to have those same functions digitally.

There are several different ways that an object can be created. You can initialize an object, as seen in one of my examples here. You can also create a constructor for an object, and instantiate an object using that constructor. Each object in JS also inherits properties and methods from the previous object type in what's called its 'prototype chain'. Each object has a private property that holds a link to its prototype, leading back down the chain to the base 'Object' that exists in JS. Along that chain, it inherits all of the methods and properties of the objects that go before it. I have an example as well demonstrating that inheritance.

I also have a few examples of methods (also called functions) here. I can include more if needed. There are a few different creation methods, as well as a method that returns all the info from an object. JSON Stringify and Parse:

test

The JSON Stringify and Parse methods are extremely helpful when relaying data from an app to a server and vice versa. Servers don't particularly like when you send javascript objects to them, so the JSON stringify method was created to rectify that situation. It converts an object into a string. The server can then use the JSON Parse method to convert that string back into an object if needed. It also lets the server send strings to the application, which can use the Parse method to convert that string into an object. It makes data transfer between an application and a server streamlined and incredibly easy to do.
